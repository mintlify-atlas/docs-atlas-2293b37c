## Using Deployed Contracts

To interact with a deployed COWFeeModule instance:

```javascript
import { ethers } from "ethers";

// Contract ABI (minimal)
const ABI = [
  "function drip(address[] calldata _approveTokens, tuple(address token, uint256 sellAmount, uint256 buyAmount)[] calldata _swapTokens) external",
  "function approve(address[] calldata _tokens) external",
  "function revoke(tuple(address token, address spender)[] calldata _revocations) external",
  "function nextValidTo() external view returns (uint32)",
  "function keeper() external view returns (address)",
  "function receiver() external view returns (address)",
  "function minOut() external view returns (uint256)"
];

// Connect to deployed contract
const provider = new ethers.JsonRpcProvider("https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY");
const signer = new ethers.Wallet(KEEPER_PRIVATE_KEY, provider);

// Ethereum Mainnet address
const moduleAddress = "0x8429bf5bad17925c364c8c30a3c2a1e0d76602d2";
const module = new ethers.Contract(moduleAddress, ABI, signer);

// Call functions
const nextValidTo = await module.nextValidTo();
console.log(`Next order valid until: ${new Date(nextValidTo * 1000)}`);
```

## CoW Protocol Settlement Contract

All COWFeeModule deployments interact with the CoW Protocol GPv2Settlement contract, which has the same address across all supported networks:

**Settlement Contract:** `0x9008D19f58AAbD9eD0D60971565AA8510560ab41`

This address is used during deployment as the `SETTLEMENT` environment variable.

## Verification Status

All deployed contracts are verified on their respective block explorers. You can view the source code, read contract state, and verify the deployment parameters directly on the block explorer.

## Network Support

The COWFeeModule is compatible with any EVM-compatible network where CoW Protocol is deployed. If you need to deploy to a network not listed here, follow the [deployment guide](/contracts/deployment-script).

## Source Code

The source code for all deployments is identical and available in the project repository:

- **Contract:** `src/COWFeeModule.sol`
- **Interfaces:** `src/interfaces/`
- **Libraries:** `src/libraries/`
- **Deployment Script:** `script/DeployCOWFeeModule.s.sol`

## Support and Questions

If you have questions about deployed contracts or need assistance:

1. Check the contract source code on the block explorer
2. Review the [contract reference](/contracts/cowfeemodule) documentation
3. Verify your configuration matches the deployment parameters
4. Open an issue in the project repository for deployment-specific questions
